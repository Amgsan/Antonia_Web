Tipos de <input>

El elemento <input> puede mostrarse de múltiples formas, dependiendo del tipo de atributo.

Si se omite el atributo type, el valor por defecto es type: "text".

Tipos de input Type estándar:

  • <input type="text">
  • <input type="password">
  • <input type="submit">
  • <input type="reset">
  • <input type="radio">
  • <input type="checkbox">
  • <input type="button">

Tipos de input Type añadidos por HTML5:

  • <input type="color"> - compatible en todos los navegadores
  • <input type="number"> - compatible en todos los navegadores
  • <input type="file"> - compatible en todos los navegadores
  • <input type="date">
  • <input type="datetime-local">
  • <input type="email">
  • <input type="hidden">
  • <input type="image">
  • <input type="month">
  • <input type="range">
  • <input type="search">
  • <input type="tel">
  • <input type="time">
  • <input type="url">
  • <input type="week">

Atributos para el elemento input

- Value: Indica el valor inicial del campo

- Readonly: campo de solo lectura

- Disabled: desactivado

- Size: tamaño del campo en caracteres

- Maxlength: tamaño máximo permitido en caracteres. No avisa al usuario si escribe caracteres de más. Para avisar habría que hacerlo con javascript.

HTML5 ha añadido otros atributos para el elemento <input>:

  • autocomplete: se puede poner a nivel de todo el formulario, o en los input de los tipos: text, search, url, tel, email, password, datepickers, range, and color
  • autofocus: indica el campo que tiene el foco al cargar la página
  • form: para indicar fuera del formulario a que formulario pertenece un campo. Hay que darle un id al formulario para poder referenciarlo
  • formaction: indica la url del fichero que procesará el campo de entrada cuando se envíe el formulario. Sobreescribe el atributo action del formulario. Se utiliza en los campos de tipo submit e image.
  • formenctype: indica el tipo de codificación a utilizar al enviar el formulario. Sólo se usa con el método post. Sobreescribe el atributo enctype del formulario. Se utiliza en los campos de tipo submit e image.
  • formmethod: indica el método HTTP para enviar los datos del formulario.Sobreescribe el atributo method del formulario. Se utiliza en los campos de tipo submit e image.
  • formnovalidate: sobreescribe el atributo novalidate del formulario. Se utiliza en los campos de tipo submit.
  • formtarget: Indica donde sacar la respuesta recibida una vez enviado el formulario. Sobreescribe el atributo target del formulario. Se utiliza en los campos de tipo submit e image.
  • height and width: para imágenes
  • list: para elementos datalist
  • min and max: para input de tipo: number, range, date, datetime-local, month, time and week
  • multiple: indica que el usuario puede indicar más de un valor
  • pattern (regexp): patrón de entrada de datos. Se usa en tipos: text, search, url, tel, email, and password.
  • placeholder: indica el valor esperado del campo. Se usa en los tipos: text, search, url, tel, email, and password.
  • required
  • step: indica los intervalos de un elemento de entrada. Se usa con tipos: number, range, date, datetime-local, month, time and week.

y atributos para el elemento <form>:

  • novalidate
