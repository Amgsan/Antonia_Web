Elemento <input>

El elemento <input> puede ser de muchos tipos, según se defina el atributo type.

Algunos ejemplos:

Type Description
<input type="text"> Defines a one-line text input field
<input type="radio"> Defines a radio button (for selecting one of many choices)
<input type="submit"> Defines a submit button (for submitting the form)

Atributo Action

El atributo action define la acción a realizar cuando se envía el formulario.

Normalmente, los datos del formulario se envían a una página web del servidor cuando el usuario pulsa el botón submit. El servidor tendrá un script que procesará los datos del formulario.

Atributo Target

Especifica si el resultado de enviar el formulario se abre en una nueva ventana en blanco del navegador, en un frame o en la ventana actual. Opciones:

1. "_self" es el valor por defecto, significa que se abrirá en la ventana actual.

2. "_blank" para que lo abra en otra ventana en blanco

3. "_parent"

4. "_parent"

5. nombre que representa el nombre de un iframe

<form action="/action_page.php" target="_blank">

Atributo Method

El atributo method especifica el método HTTP (GET o POST) a utilizar para enviar los datos del formulario. Por defecto se envía con el método GET.

Cuando se usa GET, los datos enviados serán visibles en el campo de dirección de la página:

Ejemplo: /action_page.php?firstname=Mickey&lastname=Mouse

Notas sobre GET:

  • Añade los datos del formulario en la URL como nombre/valor
  • La longitud de la URL está limitada (alrededor de 3000)
  • No utilizar nunca GET para enviar datos sensibles
  • Utilizarlo para enviar formularios cuando el usuario quiere marcar el resultados
  • GET es mejor para datos no seguros, como búsquedas en Google

Cuando usar POST?

Usarlo siempre cuando el formulario contiene datos sensibles o de información personal. The POST method does not display the submitted form data in the page address field.

Notas sobre POST:

  • POST no tiene limitaciones de espacio, y puede usarse para enviar grandes cantidades de datos.
  • El envío con POST no se puede marcar

Atributo Name

Todo campo input debe tener un atributo name para poder ser enviado. SI se omite, el dato del campo no será enviado.

Todos los atributos de la etiqueta <form>:

Attribute Description
accept-charset Specifies the charset used in the submitted form (default: the page charset).
action Specifies an address (url) where to submit the form (default: the submitting page).
autocomplete Specifies if the browser should autocomplete the form (default: on).
enctype Specifies the encoding of the submitted data (default: is url-encoded).
method Specifies the HTTP method used when submitting the form (default: GET).
name Specifies a name used to identify the form (for DOM usage: document.forms.name).
novalidate Specifies that the browser should not validate the form.
target Specifies the target of the address in the action attribute (default: _self).
